7. What is diff betn a private and protected modifier?

	Used for encapsulation and hiding sensitive data.
	Private methods cannot be overridden.

    }

-> Protected modifier allow members accessible within the same class and its subclasses even in diff pkg.

    }
}

class B extends A {

        display();                 // allowed
    }
}

8. What is constructor overloading in java?
-> Constructor Overloading means creating multiple constructors in the same class with the same name but different parameters.
ex:
class Employee {

    int id;
    String name;
    String dept;

    // 1. No-arg constructor
    Employee() {
        this.id = 0;
        this.name = "Unknown";
        this.dept = "General";
    }

    // 2. Constructor with 1 parameter
    Employee(int id) {
        this.id = id;
    }

    // 3. Constructor with 2 parameters
    Employee(int id, String name) {
        this.id = id;
        this.name = name;
    }

    // 4. Constructor with 3 parameters
    Employee(int id, String name, String dept) {
        this.id = id;
        this.name = name;
        this.dept = dept;
    }
}
Here, the class Employee has four constructors, each providing a different way to initialize an object.

10 . What is the use of super keyword in java?
-> Super keyword is used to access parent class (superclass) members such as:
	parent variables
	parent methods
	parent constructors
Mainly used in inheritance.
super keyword has 3 main uses
1) Useful when parent and child have the same variable names.
ex:
class A {
	int val = 100; }
class B extends A{
	int val = 200;
	
	void show(){
	Syso (this.value); // child value ‚Üí 200
	Syso (super.value); // parent value ‚Üí 100
} }
2) To call parent class methods
used when child class overrides a method but still wants to use parent method.
ex:
class A {
	void msg () {
		Syso ("Msg from parent");
		}
	}
class B extends A {
	@Override
	void msg (){
		super.msg(); //call parent method here
		Syso ("msg from child"); //overidin method
		}
	}
3) To call parent class constructor
must be the first statement inside the child constructor.
ex:
class A {
	A() {
		Syso ("Parent constructor");
		}
	}
class B extends A {
	B() {
		super(); //calls parent constructor
		syso ("child constructor");
		}
	}

6. What is diamond problem in java and how is it resolved?
-> A class inherits from two parent classes, and both parent classes have a method with the same name.
      A
     / \
    B   C
     \ /
      D
class B and C inherit from A.
class D inherits from class B & C.
if A has method sayHello(), and both class B & C inherit it.
then class D gets two copies of same method sayHello().
so the compiler get confused: which sayHello() should class D use - from B or from C?
This ambiguity is called the Diamond problem.

Why Java Does NOT Have the Diamond Problem?
Because Java does not support multiple inheritance with classes.
‚ùå You cannot do this in Java:
class B extends A, C { } 

Diamond problem with Java Intefaces:
interface A {
	default void show() {
		Syso("A show");
		}
	}
interface B {
	default void show() {
		Syso("B show");
		}
	}

class C implemets A, B {

	@Override
	public void show () {
		Syso("resolved show in C");
		}
	}
üëâ Java forces class C to override show() to avoid ambiguity.
